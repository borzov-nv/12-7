# Домашнее задание к занятию «Репликация и масштабирование. Часть 2 - Евгений Борзов»

### Задание 1

Опишите основные преимущества использования масштабирования методами:

- активный master-сервер и пассивный репликационный slave-сервер; 
- master-сервер и несколько slave-серверов;

Чем больше слейвов, тем выше отказоустойчивость реплик, а также способность системы обрабатывать запросы на чтение. Имеет смысл для систем, где запросы на чтение сильно превышают запросы на запись (интернет-магазин, где заказы делает небольшой процент посетителей, а большинство просто просматривает товары). Также дает возможность выделять отдельные реплики для специфических неосновных задач, таких как аналитика.

### Задание 2


Разработайте план для выполнения горизонтального и вертикального шаринга базы данных. База данных состоит из трёх таблиц: 

- пользователи, 
- книги, 
- магазины (столбцы произвольно). 

Опишите принципы построения системы и их разграничение или разбивку между базами данных.

*Пришлите блоксхему, где и что будет располагаться. Опишите, в каких режимах будут работать сервера.* 

Таблицу с магазинами можно не разделять, т.к. она не может быть большой. Пользователей можно шардировать вертикально: основные данные пользователей: ид, фио, адрес - на одном сервере, а дополнительные: история заказов, оценки, отзывы - на другом.
Книги логичнее шардировать горизонтально, например, по наличию в том или ином магазине.
